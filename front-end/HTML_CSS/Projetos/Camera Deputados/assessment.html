Minha esperiência

No geral achei a discisplina de HTML e CSS muito tranquila. Durante o percurso de aprendizagem não me deparei com dificuldades e a professora conseguiu desenvolver bem todo o conteudo com uma curva de aprendizagem muito suave.

Porém senti falta de trabalharmos mais em cima de reponsividade, visto que o mercado hoje tem como foco o mobile first, uma abordagem mais prática e construções em tempo real de sites respnsivos com nav bar e a interação de diversos outros componentes com a responsividade ficou em falta.

É percebitível que o conteúdo de CSS não se aprofundou, muito provável por causa do tempo, porém isto deixou muito a desejar. Conceitos como os de animação entre outros, muitos importantes, foram deixados de fora do conteúdo programático. Acho que valeria dividir a materia entre HTML com Interação Humano Computador e deixar 3 meses somente com CSS para termos maior profundidade de conteudo.

Em resumo, o conteudo foi bem aplicado mas poderia ter sido melhor em questão de pratica real de elaboração de sites e senti uma necessidade por mais tempo para aprofudamento do conteudo de CSS.

Design Responsivo

Questão 1

Quando é recomendando a criação de um site separado para mobile? Seria este o caso deste site da Câmara de Deputados?

resposta:

È recomendado a criação de sites serpado para mobile quando se tem grande quantidade de informações e complexidades em sites, se tornando inviável a responsividade uma vez que se faz necessário a facilitação da navegação para o site se tornar usuál.

Este não seria o caso do site da Câmara pois o site é de simples navegação não tendo interações complexas com o usuário e apenas dispondo informação.

Questão 2

No caso de um CSS específico para impressão (print), como ele deve ser inserido?

resposta:

Ele deve ser inserio através de um arquivo .css com estilos específicos para impressão e chamadno no head ou se utilizando de media type direto no css usual.

Multimídia Web

Questão 1

O elemento audio permite como elemento - filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento audio e quando devem ser utilizados?

resposta:

O source permite os atributos de midia, src e type.

Os atributos do elemento audio são autoplay, controls, loop, muted, preload, src. Ele deve ser utilizado quando se deseja inserir player de audio em sua pagina .html

Questão 2

O elemento video permite como elemento - filho o elemento source. Quais são os atributos que o source permite e qual conteúdo eles suportam? Quais os outros atributos do elemento video e quando devem ser utilizados?

resposta:

O source permite os atributos de midia, src e type.

Sobre os outros elemento do video são e servem para definir:

  • autoplay: Para dar play automático no video.

  • autobuffer: Faz o video carregar automático mesmo que não especificado para tocar automaticamente.

  • buffered: Serve para determinar os intervalos do vídeo que já foram carregados.

  • controls: Gecko oferecerá controles para permitir o usuário controlar a reprodução do vídeo, incluindo volume, navegação, e pausa/continuação da reprodução.

  • height: Para altura da área de exibição do vídeo

  • loop: Se especificado, ao chegar no fim do vídeo, ele voltará automaticamente para o começo

  • muted: Indica a configuração padrão do áudio contido no vídeo. Se definido, o áudio vai começar mudo.

  • played: TimeRanges indicando que todo o vídeo foi reproduzido.

  • preload: Esse atributo enumerado pretende dar uma sugestão ao navegador sobre o que o autor pensa que proporcionará uma melhor experiência do usuário.

  • poster: Uma URL indicando uma imagem de prévia do vídeo até o usuário reproduzir ou navegar por ele.

  • src: A URL do vídeo a ser incorporado.

  • width: A largura da área de exibição do vídeo
